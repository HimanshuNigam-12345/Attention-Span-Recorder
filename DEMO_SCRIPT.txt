[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's walk through the architecture of the system. 
First, when a user enters a query, it passes through the Security Checkpoint node which scrubs PII, blocks prompt injections, and logs audit metrics.
If passed, the query goes to the Orchestrator node, which coordinates between our specialized Distraction Analyst and Gamification Coach sub-agents.
For actions like redeeming hard-earned focus coins, the flow routes to the Human-in-the-Loop approval node, which pauses execution to confirm the transaction.
Finally, the terminal node formats the response and appends our simulated Productivity Dashboard.
(pause)
Additionally, both specialized sub-agents are equipped with an MCP Toolset, allowing them to access external distraction definitions, study logs, and coaching tips locally.

The system also runs a local stdio MCP server, separating the core agent coordination from our local tool implementations.
